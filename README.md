# Dashboard & Counting Service — AWS Terraform Deployments

This repository contains two versions of the same infrastructure: a **Dashboard** service and a **Counting** service deployed on AWS EC2 using Terraform. Both versions deploy the [HashiCorp demo-consul-101](https://github.com/hashicorp/demo-consul-101) microservices.

---

## Architecture Overview

```
Internet
    │
    ▼
[ Dashboard EC2 ] ──── Public Subnet (10.x.x.0/25)
    │  port 80 (public)      │
    │  port 22 (public)   IGW / NAT GW
    │                        │
    ▼                        ▼
[ Counting EC2 ] ──── Private Subnet (10.x.x.128/25)
    port 80 (VPC only)
    port 22 (VPC only)
```

- **Dashboard** runs on a **public subnet** — accessible from the internet on port 80
- **Counting** runs on a **private subnet** — only reachable from within the VPC
- Dashboard connects to Counting via private IP (`http://<counting-private-ip>:80`)
- SSH key pair is auto-generated by Terraform using the `tls` provider (ED25519)

## Versions

### V1 — `dashboard-accounting-AWS` (Plain Terraform Resources)

Builds all infrastructure using **native AWS Terraform resources** — no community modules. Every resource (VPC, subnets, IGW, NAT GW, route tables, security groups, EC2 instances) is defined explicitly.

**File structure:**

```
dashboard-accounting-AWS/
├── vpc.tf               # VPC, subnets, IGW, NAT GW, route tables, security groups
├── instance.tf          # EC2 instances, TLS key, key pair
├── variables.tf         # Input variables with defaults
├── outputs.tf           # Dashboard public IP
└── shell_scripts/
    ├── dashboard.sh     # Installs & starts dashboard-service as a systemd unit
    └── counting.sh      # Installs & starts counting-service as a systemd unit
```

**Default variable values:**

| Variable | Default |
|---|---|
| `vpc_cidr` | `10.100.0.0/24` |
| `dashboard_subnet_cidr` | `10.100.0.0/25` |
| `counting_subnet_cidr` | `10.100.0.128/25` |
| `dashboard_private_ip` | `10.100.0.100` |
| `counting_private_ip` | `10.100.0.200` |
| `instance_type` | `t2.micro` |

**Deploy:**

```bash
cd dashboard-accounting-AWS
terraform init
terraform apply
```

### V2 — `dashboard-accounting-AWS-V2` (Official Terraform Modules)

Same architecture as V1 but uses **official public Terraform modules** from the [terraform-aws-modules](https://registry.terraform.io/namespaces/terraform-aws-modules) organization. Variable values are provided via `terraform.tfvars`.

**Modules used:**

| Module | Version | Purpose |
|---|---|---|
| `terraform-aws-modules/vpc/aws` | 6.6.0 | VPC, subnets, IGW, NAT GW, route tables |
| `terraform-aws-modules/security-group/aws` | 5.3.1 | Dashboard & counting security groups |
| `terraform-aws-modules/ec2-instance/aws` | 6.4.0 | Dashboard & counting EC2 instances |

**File structure:**

```
dashboard-accounting-AWS-V2/
├── terraform.tf         # Module definitions + TLS key + key pair resource
├── terraform.tfvars     # Variable values
├── variables.tf         # Input variable declarations
├── outputs.tf           # Dashboard public IP
└── shell_scripts/
    ├── dashboard.sh     # Same as V1
    └── counting.sh      # Same as V1
```

**`terraform.tfvars` values:**

| Variable | Value |
|---|---|
| `vpc_cidr` | `10.0.0.0/24` |
| `public_subnets` | `["10.0.0.0/25"]` |
| `private_subnets` | `["10.0.0.128/25"]` |
| `dash_priv_ip` | `10.0.0.100` |
| `count_priv_ip` | `10.0.0.200` |
| `instance_type` | `t2.micro` |
| `ami_id` | `ami-0f8faa29480e7e6de` |
| `azs` | `["ap-northeast-1a"]` |

**Deploy:**

```bash
cd dashboard-accounting-AWS-V2
terraform init
terraform apply
```

## V1 vs V2 Comparison

| | V1 (Plain Resources) | V2 (Modules) |
|---|---|---|
| **Approach** | All resources written manually | Official community modules |
| **Lines of config** | More verbose | More concise |
| **Variables** | Defaults in `variables.tf` | Values in `terraform.tfvars` |
| **EIP for Dashboard** | `associate_public_ip_address = true` | `create_eip = true` (dedicated EIP) |
| **Learning value** | High — shows every underlying resource | High — shows real-world module usage |
| **Module dependencies** | None | 3 public modules |

## Prerequisites

- Terraform >= 1.0
- Sufficient IAM permissions to create VPC, EC2, EIP, NAT Gateway, and key pair resources

## Output

Both versions output the dashboard's public IP after apply:

```
dashboard_public_ip = "<public-ip>"
```

Access the dashboard at: `http://<dashboard_public_ip>`
